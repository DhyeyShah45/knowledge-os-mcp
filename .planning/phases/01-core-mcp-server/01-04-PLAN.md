  <action>
    Add these imports near the top of server.py if not already present: `import frontmatter`, `from datetime import datetime, date`.

    Append the following four `@mcp.tool()` async functions to server.py (place them after OAuth endpoints, before `app.mount("/mcp", ...)` if not already mounted; if already mounted, place them anywhere above mount — `@mcp.tool()` registers with the FastMCP instance regardless of source position):

    1. `async def list_folders() -> dict` (NAV-01): walk only the `wiki/` subdirectory of VAULT_PATH. For each directory (including wiki/ itself), count direct-child `*.md` files (non-recursive count). Return `{"folders": [{"path": rel_path_str, "note_count": int}, ...]}` sorted by path. The rel_path_str is the path relative to VAULT_PATH, using forward slashes (e.g. `"wiki/entities"`). Skip hidden directories (names starting with `.`). If `wiki/` does not exist (vault not initialized), return `{"folders": []}`. No path traversal risk here because no user input.

    2. `async def list_notes(folder: str = None) -> dict` (NAV-02): if `folder` is None, default to `"wiki"`. Validate via `safe_vault_path(folder)` — return the error dict if invalid. Verify the resolved Path is a directory; if not, return `err(f"Not a directory: {folder}", ERR_NOT_FOUND)`. List direct-child `*.md` files. For each note: read frontmatter via `post = frontmatter.load(str(p))`. Title resolution order: `post.get("title")` → fall back to filename stem (e.g. `"transformer-architecture"`). `last_modified` = `date.fromtimestamp(p.stat().st_mtime).isoformat()`. Return `{"notes": [{"path": rel_path_str, "title": ..., "last_modified": "YYYY-MM-DD"}, ...]}` sorted by path. Skip files that fail to parse (catch broad exception, log to nothing — silently skip).

    3. `async def get_note_metadata(path: str) -> dict` (NAV-03): validate via `safe_vault_path(path)`; return error if invalid. Verify the resolved Path exists and is a file with `.md` extension; if not, return `err(f"Note not found: {path}", ERR_NOT_FOUND)`. Load with python-frontmatter. Build response: `title` from `post.get("title", p.stem)`; `date` from `post.get("date", "")` (coerce to ISO string if a date/datetime); `tags` from `post.get("tags", [])`; `sources` from `post.get("sources", [])`; `related` from `post.get("related", [])`; `word_count = len(post.content.split())`; `last_modified = date.fromtimestamp(p.stat().st_mtime).isoformat()`. Do NOT include body content in the response (the entire point of NAV-03 is to skip body reads).

    4. `async def get_index() -> dict` (NAV-04): construct `index_path = VAULT_PATH / "wiki" / "index.md"`. If missing, return `err("Index not found — has init_vault.py run?", ERR_NOT_FOUND)`. Read content with `index_path.read_text(encoding="utf-8")`. Parse the header line (second line of file): expected format `Last updated: YYYY-MM-DD | Total pages: N | Total sources: N`. Use regex `r"Last updated:\s*(\S+)\s*\|\s*Total pages:\s*(\d+)\s*\|\s*Total sources:\s*(\d+)"` to extract values; if regex does not match (e.g. user-edited index), fall back to defaults `last_updated=""`, `total_pages=0`, `total_sources=0`. Return `{"content": <full file content>, "total_pages": int, "total_sources": int, "last_updated": str}`.

    Tie-breaking and edge cases:
    - All tools must catch FileNotFoundError, PermissionError on any I/O and return the standard error dict (do not let exceptions propagate to MCP layer per D-08).
    - The tools return plain dicts — FastMCP serializes them as MCP tool results automatically.
