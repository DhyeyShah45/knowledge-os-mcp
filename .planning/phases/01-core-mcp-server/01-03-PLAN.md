<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet (claude.ai mobile) → /authorize, /token | Public unauthenticated endpoints (OAuth flow); validated by client_id+redirect_uri allowlist + PKCE |
| Internet → /mcp/* | Bearer-token-gated; requires successful OAuth flow OR direct VAULT_SECRET (Claude Desktop path) |
| /mcp request → tool function | Path parameters from MCP clients flow into tool functions; future tools must call safe_vault_path |
| Server process → VAULT_SECRET env var | Plaintext in memory; never logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Missing Authorization header on /mcp | mitigate | BearerAuthMiddleware returns 401 with AUTH_ERROR dict before request hits MCP layer |
| T-03-02 | Spoofing | OAuth client_id forgery | mitigate | /authorize compares client_id to env var OAUTH_CLIENT_ID; reject otherwise |
| T-03-03 | Tampering | OAuth code replay | mitigate | _pkce_codes tracks `used: bool`; /token marks code used and rejects reuse |
| T-03-04 | Tampering | OAuth code substitution (without verifier) | mitigate | PKCE S256 challenge/verifier hash compare; reject on mismatch |
| T-03-05 | Tampering | Path traversal via future tool path parameters | mitigate | safe_vault_path() returns INVALID_PATH dict if resolved path escapes VAULT_PATH (CONTEXT.md decision 5) |
| T-03-06 | Tampering | Modification of vault raw/ directory by future tools | mitigate | safe_vault_path rejects paths whose first component is `raw` |
| T-03-07 | Information Disclosure | VAULT_SECRET in logs | mitigate | BearerAuthMiddleware never logs the Authorization header (no `print`/`logger` calls on it) |
| T-03-08 | Information Disclosure | OAuth code in browser history | accept | Codes are single-use and short-lived; standard OAuth tradeoff |
| T-03-09 | Denial of Service | Unbounded _pkce_codes growth | accept | In-memory dict; single-user system; acceptable for v1. Documented as v2 hardening. |
</threat_model>
