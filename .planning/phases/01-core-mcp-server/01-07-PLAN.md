<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → category/operation args | User-controlled strings; allowlist-validated |
| MCP client → summary/notes/title args | Free-form content written to vault index/log |
| Internet → Cloudflare Tunnel → localhost:8000 | Production traffic path; auth enforced upstream by BearerAuthMiddleware |
| PM2 → uvicorn process | Process supervision; restart-on-crash is the design |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Unknown category injecting arbitrary section into index.md | mitigate | update_index validates category against allowlist {entities, concepts, sources, queries} |
| T-07-02 | Tampering | Unknown operation polluting log.md format | mitigate | append_log validates operation against allowlist {ingest, query, lint, update} |
| T-07-03 | Information Disclosure | log.md leaking sensitive notes via tunnel | accept | log entries are author-controlled; bearer-token-gated; single-user system |
| T-07-04 | Denial of Service | Adversary spamming /mcp on the public tunnel URL | accept | Cloudflare provides basic rate limiting at the edge; v2 may add per-token rate limits |
| T-07-05 | Spoofing | OAuth client_id leak enabling someone else to obtain VAULT_SECRET via /token | mitigate | PKCE verifier prevents code interception attacks; redirect_uri allowlist limits attack surface |
</threat_model>
