<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → path arg on every ingestion tool | User-controlled file path |
| MCP client → content/text args | Arbitrary bytes written to disk |
| MCP client → frontmatter dict | User-controlled metadata; YAML serialization handled by python-frontmatter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | Path traversal on any ingestion tool | mitigate | All five tools call safe_vault_path on path arg |
| T-06-02 | Tampering | Write into raw/ (immutable per CLAUDE.md) | mitigate | safe_vault_path rejects paths starting with raw/ |
| T-06-03 | Tampering | YAML injection via frontmatter dict values | accept | python-frontmatter uses PyYAML safe_dump internally; risk is low for single-user system; values are written as-is and not executed |
| T-06-04 | Tampering | Race condition: concurrent writes to same file | accept | Phase 1 single-user; documented in RESEARCH.md Pitfall 5. asyncio.Lock per-path deferred to v2 |
| T-06-05 | Information Disclosure | append_to_note exposing file contents to other tools' returns | accept | Returns word_count only, not content |
</threat_model>
