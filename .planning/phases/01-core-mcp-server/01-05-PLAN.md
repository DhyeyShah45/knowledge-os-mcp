    Then append four `@mcp.tool()` async functions:

    1. `async def search_full_text(query: str, top_k: int = 5) -> dict` (RET-01):
       - If query is empty: return `{"results": []}`.
       - Iterate `(VAULT_PATH / "wiki").rglob("*.md")` (D-14 wiki/ scope).
       - For each file: read content; lower = content.lower(); query_lower = query.lower(); count = lower.count(query_lower) (D-13 case-insensitive, includes frontmatter per D-13).
       - If count == 0: skip.
       - Find first occurrence line index: split content into lines; find first line whose lowercase contains query_lower.
       - Build 2-line snippet (D-15): take the matching line plus the next line (or previous line if matching line is last); join with `\n`; truncate to ~150 chars if longer.
       - Title resolution: try to load frontmatter title via `frontmatter.load(str(p)).get("title")`; fall back to filename stem.
       - Collect candidates `[{"path": rel, "title": ..., "snippet": ..., "_count": count}, ...]`.
       - Sort by `_count` descending; take top_k.
       - Compute D-12 score: `max_count = max(c["_count"] for c in candidates) if candidates else 1`; each result `score = c["_count"] / max_count`. Single result yields score 1.0.
       - Strip `_count` from output; return `{"results": [...]}`.

    2. `async def get_note_summary(path: str) -> dict` (RET-02):
       - Validate via safe_vault_path; return error if invalid.
       - Verify file exists with .md extension; else err NOT_FOUND.
       - Load with frontmatter; title = post.get("title", p.stem); body = post.content.
       - summary = body[:200] (PRD §5.2 first 200 chars). If body is shorter, take all.
       - headings = [s["heading"] for s in _get_sections(body)].
       - word_count = len(body.split()).
       - Return `{"title": ..., "summary": ..., "headings": [...], "word_count": ...}`. Do NOT include full body.

    3. `async def read_note(path: str) -> dict` (RET-03):
       - Validate via safe_vault_path; return error if invalid.
       - Verify file exists; else err NOT_FOUND.
       - Load with frontmatter; title = post.get("title", p.stem); content = post.content; fm_dict = dict(post.metadata).
       - Coerce any datetime/date values in fm_dict to ISO strings so the dict is JSON-serializable.
       - Return `{"path": rel, "title": title, "frontmatter": fm_dict, "content": content}`.

    4. `async def read_note_section(path: str, heading: str) -> dict` (RET-04):
       - Validate via safe_vault_path; return error if invalid.
       - Verify file exists; else err NOT_FOUND.
       - Read content; sections = _get_sections(content).
       - idx = _find_heading_index(sections, heading). If None: return `err(f"Heading not found: {heading}", ERR_HEADING_NOT_FOUND)` augmented with `"available_headings": [s["heading"] for s in sections]` (per CONTEXT.md Specifics). Construct the dict manually so the extra field merges with the standard error shape.
       - Determine section end: find the next section index `j > idx` where `sections[j]["level"] <= sections[idx]["level"]`; if none, end = len(lines).
       - Slice lines from `sections[idx]["start_line"] + 1` (line after heading) to `sections[j]["start_line"]` (or end).
       - Join lines and strip trailing whitespace.
       - Return `{"path": rel, "heading": sections[idx]["heading"], "content": section_content}`.

    All four tools wrap their main body in try/except: any file I/O error returns `err(str(e), ERR_NOT_FOUND)` rather than raising. No HTTPException (D-08).
