# knowledge-os-mcp

A personal, persistent, compounding knowledge base built on Obsidian — maintained by Claude, accessible from any device. A FastAPI MCP server runs on a local laptop, exposed via Cloudflare Tunnel, so Claude on any device (phone, desktop, web) can read from and write to the same vault over HTTPS.

The vault is a living wiki: every ingested source, query, and operation makes the knowledge base richer. Claude handles all bookkeeping (filing, indexing, cross-referencing). You handle curation and intent.

---

## Prerequisites

- **Python 3.11+** — `brew install python@3.11`
- **Node.js 18+** — needed for PM2 process manager: `brew install node`
- **PM2** — `npm install -g pm2`
- **Homebrew** — https://brew.sh
- **cloudflared** — `brew install cloudflared`
- **Cloudflare account** with a registered domain (free tier is sufficient)

## Quick Start

### 1. Run setup.sh

```bash
./setup.sh
```

This will:
- Check Python 3.11+ is available
- Create a Python virtualenv at `.venv/`
- Install all dependencies from `requirements.txt`
- Check cloudflared is installed
- Generate `~/.cloudflared/config.yml` template (if it doesn't exist yet)

### 2. Populate .env

```bash
cp .env.example .env
```

Edit `.env` and set all four values:

```bash
VAULT_SECRET=<random-secret>       # openssl rand -hex 32
VAULT_PATH=/path/to/new/vault      # must NOT exist yet — init_vault.py creates it
OAUTH_CLIENT_ID=<your-client-id>   # any string; registered with claude.ai connector
OAUTH_REDIRECT_URI=<callback-url>  # e.g. https://vault.yourdomain.com/oauth/callback
```

> **VAULT_SECRET** serves a dual role: it is the bearer token for Claude Desktop (direct connection) and the OAuth access token issued by the `/token` endpoint for claude.ai mobile.

### 3. Initialize the vault

```bash
source .env
python init_vault.py
```

This creates the full vault directory tree at `VAULT_PATH` and seeds `wiki/index.md` and `wiki/log.md`. **Run this exactly once** — it will fail if `VAULT_PATH` already exists (preventing accidental re-initialization).

### 4. Start with PM2

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # follow the printed instructions to auto-start on boot
```

Check status with `pm2 list` and view logs with `pm2 logs vault-mcp`.

## Cloudflare Tunnel DNS

Cloudflare Tunnel exposes `localhost:8000` as a persistent HTTPS subdomain (e.g. `vault.yourdomain.com`) with no port-forwarding required. These steps cannot be scripted because they require browser authentication.

### One-time tunnel setup

```bash
# 1. Authenticate cloudflared with your Cloudflare account
cloudflared tunnel login

# 2. Create a named tunnel called "vault"
cloudflared tunnel create vault
#    This prints a UUID — copy it

# 3. Route DNS: point vault.yourdomain.com to the tunnel
cloudflared tunnel route dns vault vault.yourdomain.com

# 4. Edit the config template generated by setup.sh:
#    ~/.cloudflared/config.yml
#    Replace <TUNNEL-UUID> and <YOUR-USERNAME> with actual values
```

### ~/.cloudflared/config.yml (after editing)

```yaml
tunnel: <your-tunnel-uuid>
credentials-file: /Users/<your-username>/.cloudflared/<your-tunnel-uuid>.json

ingress:
  - hostname: vault.yourdomain.com
    service: http://localhost:8000
  - service: http_status:404
```

Once PM2 starts the `cloudflare-tunnel` process, the tunnel runs persistently.

## OAuth Connector (claude.ai Mobile)

The **claude.ai web and mobile connector** does not accept static bearer tokens — it requires OAuth 2.0 to complete a handshake. This server implements **OAuth 2.0 Authorization Code + PKCE (S256)** to satisfy that requirement.

The OAuth flow is a thin wrapper: the `/token` endpoint simply returns `VAULT_SECRET` as the access token. Since this is a single-user system, the same shared secret serves both the Claude Desktop direct-bearer path and the claude.ai mobile OAuth path.

### Connector configuration in claude.ai

In the claude.ai connector UI:
- **Server URL:** `https://vault.yourdomain.com/mcp`
- **OAuth authorization URL:** `https://vault.yourdomain.com/authorize`
- **OAuth token URL:** `https://vault.yourdomain.com/token`
- **Client ID:** the value of `OAUTH_CLIENT_ID` in your `.env`
- **Redirect URI:** the value of `OAUTH_REDIRECT_URI` in your `.env`

The redirect URI must match a URL that the claude.ai connector can receive the authorization code at (typically provided by the connector UI).

### Claude Desktop (local, no OAuth needed)

For Claude Desktop on the same machine, configure MCP directly with the bearer token:

```json
{
  "mcpServers": {
    "vault": {
      "url": "http://localhost:8000/mcp",
      "headers": {
        "Authorization": "Bearer <your-VAULT_SECRET>"
      }
    }
  }
}
```

## Troubleshooting

### PM2 process not starting

```bash
pm2 logs vault-mcp --lines 50
pm2 logs cloudflare-tunnel --lines 50
```

Common causes:
- `.venv/bin/uvicorn` not found — run `./setup.sh` to create the virtualenv
- `server.py` not yet created — server is implemented in Phase 1 Plan 02
- `.env` not populated — `VAULT_SECRET` or `VAULT_PATH` missing causes startup failure
- `VAULT_PATH` directory not yet created — run `python init_vault.py`

### Virtualenv issues

```bash
# Rebuild the virtualenv from scratch
rm -rf .venv
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Tunnel auth errors

```bash
# Re-authenticate if credentials expire
cloudflared tunnel login

# Check tunnel is listed
cloudflared tunnel list

# Test tunnel connectivity
cloudflared tunnel info vault
```

### Checking server health (after server.py exists)

```bash
# Bearer token auth (Claude Desktop path)
curl -H "Authorization: Bearer $VAULT_SECRET" http://localhost:8000/mcp

# Check PM2 process status
pm2 status
```
